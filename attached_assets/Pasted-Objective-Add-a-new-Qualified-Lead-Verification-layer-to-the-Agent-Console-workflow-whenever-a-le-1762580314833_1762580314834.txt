Objective:
Add a new “Qualified Lead Verification” layer to the Agent Console workflow whenever a lead is marked as Qualified.
This ensures every qualified lead is human-verified, AI-validated, and audit-traceable through LinkedIn or call-based confirmation.

🔹 Trigger Condition:
When a lead is marked as Qualified and submitted, the system must trigger a verification modal asking:

“Is this Lead — [Contact Name] — currently holding the position of [Job Title] at [Company Name]?”


🔹 Response Options:


✅ Yes – Lead is Present


☎️ On-Call Confirmation



🔹 If “Yes – Lead is Present” is Selected:


Display upload prompt:

“Please upload a screenshot of the Lead’s LinkedIn profile showing Name, Company, and Current Position.”



Once uploaded, the system should:


Attach the image to the lead record.


Automatically route the image file to the QA & Leads Portal for AI-powered image analysis and validation.


The AI validation module should:


Extract and analyze key text regions (Name, Company, Title).


Compare extracted data with the CRM Lead Record.


Assign a Confidence Score (0–100%) for identity and employment validation.


Return validation result flags:


✅ Verified Match


⚠️ Partial Match – Review Needed


❌ Mismatch / Invalid Proof








Upon successful submission:


System logs the verification type = LinkedIn Verified


Stores:


Uploaded file reference


AI validation report (JSON payload)


Agent ID and Timestamp




Updates lead record with verification status (e.g., “Pending QA Review” → “AI Verified” → “Approved”)





🔹 If “On-Call Confirmation” is Selected:

(Optional): Uploaded screenshot may also be passed to QA & Leads Portal for AI-assisted validation, ensuring title and company match where applicable.



🔹 QA & Leads Portal Integration (AI Validation Flow)
Input: Screenshot from Agent Console
Process:


OCR-based text extraction


AI comparison with CRM Lead Record


Confidence scoring (Name, Company, Title match %)


Fraud / mismatch detection (e.g., outdated job, unrelated company, or duplicate contact)


Output:
AI Validation JSON returned to CRM:
{
  "lead_id": "L-87213",

}

Final Statuses:


✅ Verified and Approved


⚠️ Flagged for QA Review


❌ Rejected (Invalid Proof or Mismatch)

Mandatory Verification Step before any “Qualified” submission.


Two verification paths:


LinkedIn Verification (with AI + QA validation)


On-Call Verification (manual + optional AI check)




Seamless data flow between Agent Console → CRM → QA & Leads Portal.


Full audit trail including:


Agent actions


Uploaded assets


AI analysis


Call records and timestamps
