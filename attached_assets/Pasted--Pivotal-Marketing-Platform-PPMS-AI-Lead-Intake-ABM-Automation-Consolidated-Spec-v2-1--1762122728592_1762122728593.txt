# Pivotal Marketing Platform — PPMS + AI Lead Intake & ABM Automation (Consolidated Spec v2.1)

## 0️⃣ Objective

Design and build a next‑generation **Pipeline Management Platform (PPMS)** for Pivotal B2B that fuses:

* Dynamic Pipelines with integrated Accounts and Contacts
* True Microsoft 365 OAuth Email sync (Graph API)
* **AI-driven Account & Company Intelligence**
* Automated, personalized **email/LinkedIn sequences**
* **Outlook‑safe** email templates with ABM targeting
* Modular, data-driven, GDPR/CCPA/TCPA‑compliant foundations

This document **combines**:

1. The **PPMS Full Implementation Prompt (v2)**, and
2. The **Lead Intake → CRM Completion → Enrichment → AI Analysis → Personalized Outreach** workflow for partial form submissions.

## 4️⃣ Lead Capture → CRM Completion (Partial Form Flow)

### 4.1 Flow Overview

Form (partial) → **Intake API** → Normalize/Dedupe → **Auto‑complete from CRM** (Contacts/Accounts) → **Selective Enrichment** (only if gaps) → Create/Update Lead → emit `lead.captured` event → **AI analysis** → **Personalized outreach**.

### 4.2 Minimal Form Payload → `/api/intake/lead`

Required (soft): `email`, `asset_title`, `asset_type`, `consent_timestamp`, `utm{source,medium,campaign,term,content}`
Optional: `first_name`, `last_name`, `job_title`, `company_name`, `country/state/city`

**Derived**: `company_domain` from email; `source_bucket` from asset (e.g., `Exec_ABM_Guide_Q4_2025`).

### 4.3 Intake Logic (server)

1. Normalize & extract domain
2. Dedupe: Contact (by email) → Lead
3. Auto‑link Account (domain > exact name > fuzzy+country)
4. Back‑fill Lead fields from existing Contact/Account
5. If critical gaps remain (title/industry/domain), queue **Enrichment**
6. Emit `lead.captured` for AI

### 4.4 Enrichment Queue

* Resolve domain; pull `industry`, `headcount_band`, `hq_country`, `revenue_band`, `description`, `tech_stack`; contact `job_title/seniority/phone/linkedin_url`
* Write provenance + timestamp; throttle & cache vendor calls

### 4.5 Dedupe & Merge

* Email exact → same person; personal vs corporate → keep corporate primary
* Link Lead → existing Contact where present (no duplicate contacts)
* Mark junk/test records; quarantine table for fixes

### 4.6 Compliance

* Persist `consent_timestamp`, `consent_source`, `processing_basis`; mask PII in logs; AES‑256 at rest; RBAC; audit log

## 5️⃣ AI Account Analyst (Deep Account & Company Insights)

### 5.1 Purpose

Generate structured **AccountBrief** using internal CRM data + public signals: products, positioning, markets, buyer personas, pains/triggers, **tailored outreach angles**.

### 5.2 Data Flow

* **Ingest**: account domain, description, notes; linked contacts/leads
* **Enrich**: industry normalization & product keywords
* **Synthesize**: LLM produces AccountBrief JSON
* **Cache** per Account; auto‑refresh every 14 days (or on demand)
* **Expose**: `GET /ai/accounts/:id/brief`, `POST /ai/accounts/:id/analyze`

### 5.3 Example Output

## 6️⃣ Automated Sequences (Email/Tasks/LinkedIn)

### 6.1 Capabilities

* Multi‑step sequences with delays/branches; enroll by Account/Contact
* Stop rules: reply, unsubscribe, bounce
* A/B: subject/body variants
* Real send via Graph `Mail.Send`; analytics via webhooks (open/click/reply/bounce)

### 6.2 Models

```prisma
model Sequence {
  id         String @id @default(cuid())
  name       String
  ownerId    String
  steps      Json    // [{id, kind:'email'|'task'|'linkedin', delay:{days}, templateId, branch}]
  targetType String  // 'account' | 'contact'
  status     String  // 'draft' | 'active' | 'paused'
}

model SequenceEnrollment {
  id           String @id @default(cuid())
  sequenceId   String
  targetId     String
  status       String  // 'queued'|'in_progress'|'paused'|'completed'
  currentStep  String?
  lastEventAt  DateTime?
}
```

### 6.3 Personalization Tokens

```
{{contact.firstName}}, {{contact.jobTitle}}, {{account.name}}, {{account.industry}},
{{ai.openingHook}}, {{ai.tailoredOffer}}, {{ai.caseStudy}}, {{rep.signature}}
```
