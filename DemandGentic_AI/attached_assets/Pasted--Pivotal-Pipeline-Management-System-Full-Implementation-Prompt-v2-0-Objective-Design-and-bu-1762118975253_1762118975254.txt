🧠 Pivotal Pipeline Management System — Full Implementation Prompt (v2)
0️⃣ Objective

Design and build a next-generation Pipeline Management Platform (PPMS) for Pivotal B2B, combining:

Dynamic Pipelines with integrated Accounts and Contacts

True Microsoft 365 OAuth Email sync (Graph API)

AI-driven Account Intelligence

Automated Email Sequences with personalization

Outlook-safe email templates with ABM targeting

The system must be modular, data-driven, and enterprise-compliant (GDPR, CCPA, TCPA).

2️⃣ Core Functional Modules
2.1 Pipeline Management

Full CRUD for Pipelines, Stages, and Deals

Drag-and-drop Kanban UI (React DnD / dnd-kit)

Inline editing for deal value, owner, stage, close date

Stage transitions auto-save with optimistic UI updates

Pipeline metrics: total value, win rate, forecast by month

2.2 Account & Contact Integration

Accounts link directly to associated Contacts and Deals

Selecting an Account in Deal Creation filters related Contacts

Account view includes:

Open/closed deals

Contacts with activity score

AI-generated AccountBrief (see section 4)

Email & meeting timeline synced via M365

3️⃣ Microsoft 365 Email Integration (Graph API)
3.1 OAuth 2.0 Flow

Use authorization_code + PKCE flow

Scopes: offline_access, Mail.ReadWrite, Mail.Send, Calendars.Read, User.Read

Store encrypted tokens (access + refresh) in DB

Auto-refresh access tokens; fail gracefully if expired

Connection UI shows status + “Reconnect” button

3.2 Email & Calendar Sync

Fetch inbound/outbound messages tied to known domains

Log as Activity objects (email, meeting, task)

Sync scheduler runs every 6 hours; on-demand via /m365/sync

Respect GDPR: store only metadata & bodyPreview (not full email body)

3.3 Reply Detection

Graph Webhook subscription for /me/messages

Auto-detect replies → mark sequence completed

4️⃣ AI Account Analyst (Deep Account Insights)
4.1 Purpose

Generate structured account intelligence using internal + public data:

Main products & positioning

Target industries & geos

Buyer titles (decision-maker personas)

Key pains & triggers

Tailored outreach angles

4.2 Data Flow

Ingest: Account domain + company description + manual notes

Enrich: Industry normalization, product keywords

Synthesize: AI model generates AccountBrief JSON

Cache: Version per Account; refresh every 14 days

Expose: /ai/accounts/{id}/brief and /ai/accounts/{id}/analyze

5️⃣ Automated Email Sequences
5.1 Capabilities

Multi-step email & task sequences with delays and conditions

Enrollment by Account or Contact list

Pause on reply, unsubscribe, or bounce

A/B test (subject + body variants)

Use M365 Graph Mail.Send for real sending

5.2 Sequence Model
model Sequence {
  id          String @id @default(cuid())
  name        String
  ownerId     String
  steps       Json    // [{id, kind:'email'|'task', delay:{days}, templateId, branch}]
  targetType  String  // 'account' | 'contact'
  status      String  // 'draft' | 'active' | 'paused'
}

model SequenceEnrollment {
  id          String @id @default(cuid())
  sequenceId  String
  targetId    String
  status      String  // 'queued'|'in_progress'|'paused'|'completed'
  currentStep String?
  lastEventAt DateTime?
}

5.3 Send Engine

Cron/queue picks due sends

Token substitution from Account/Contact/AI data

M365 Send via /me/sendMail

Stop rules: reply, bounce, unsubscribe

A/B logic: randomized variant per enrollment

Analytics: open, click, reply, bounce tracked via webhook
